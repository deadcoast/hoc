## RRC — Release Readiness Checklist (0.0.4 “Concrete and Solid”)

## RRC.1 Purpose

Define objective gating criteria to declare CAH 0.0.4 ready for publication and as the authoritative state of truth for subsequent backports from 0.0.1/0.0.2.

## RRC.2 Governance & Roles

* **Spec Owner (SO):** accountable for final sign-off.
* **Editor (ED):** integrates changes, maintains diagrams.
* **Validator Lead (VL):** maintains validator implementation and CTM/TOS conformance.
* **Registry Steward (RS):** manages CBS/IRS/CRL/SAS/RPP artifacts.
* **Security Reviewer (SR):** reviews hashing, signatures, transport.

## RRC.3 Hard Gates (MUST PASS)

* [ ] **Diagram supremacy**: RCA round-trip equality holds; `diagram_digest` stable.
* [ ] **Grammar completeness**: GRAMMAR-CORE and NODE-IO, SPAN-OBJECT finalized; no TBD sections.
* [ ] **Operator legality**: OP-LEG-SCHEMA and OP-LEG-RULESET published; P-classes mapped to diagram.
* [ ] **C-SP routing**: CSP-ROUTING finalized with port definitions (P_E1/P_P1/P_M1/P_A1).
* [ ] **Serialization**: schema validated; canonical ordering confirmed via golden outputs.
* [ ] **Validation suite**: VALIDATION-SUITE rules implemented and align with the phase graph.
* [ ] **Parity thresholds**: T3 defaults (`θ_emo=1.00`, `θ_phl=1.00`, `ρ=0.67`) pinned in PARITY-NORM-IFACE.
* [ ] **Error dictionary**: ERROR-CATALOG exhaustive for current scope; codes unique.
* [ ] **Whitespace/layout**: numeric constraints enforced (triple-v spacing, five-v fence).
* [ ] **Timing & concurrency**: tie-break matrix implemented.
* [ ] **Conformance harness**: CTM and TOS schemas validated with a minimum of:

  * 2+ tests per category LEX…DEF, and at least one PERF.
* [ ] **Security**: RCA hashing (SHA-256), CBS/IRS/CRL/SAS signatures verified end-to-end.
* [ ] **RBM**: Reference bundle produced and self-verifies (`bundle_digest`).

## RRC.4 Soft Gates (SHOULD PASS; WARN if unmet)

* [ ] **Portability profile**: RPP prepared for at least one implementation (LEVEL_B+).
* [ ] **Compliance badge**: CBS issuance validated with a test badge (non-production).
* [ ] **Issuer registry**: IRS authored and signed by RA; expiry ≥ 30 days from release.
* [ ] **Status API**: SAS endpoint mock validated by BVP.
* [ ] **Audit logging**: VAL integrated with basic integrity chaining.

## RRC.5 Documentation

* [ ] **Spec Index** (SI) included in RBM.
* [ ] **Changelog** (VCT) entry created with compatibility tag (`backward|breaking`).
* [ ] **Test Index** (CTM-INDEX) summarizing coverage and known gaps.

## RRC.6 Backport Plan (0.0.1/0.0.2 → 0.0.4)

* [ ] Publish a **migration note** in VCT:

  * Keyword/identifier deltas,
  * Operator legality changes,
  * Serialization diffs,
  * Error code remaps (legacy → current).
* [ ] Provide **adapters** (if any) to coerce older serialized forms; flag with `E-VERSION-901` WARN where feasible.
* [ ] Confirm 0.0.1/0.0.2 diagrams can be normalized to 0.0.4 or are explicitly unsupported.

## RRC.7 Release Artifacts

* [ ] **Spec bundle** (RBM) signed; `bundle_digest` recorded in VCT and CBS examples registry.
* [ ] **Conformance results** (TOS) archived; link/digest embedded in badge evidence for Level C reference validator.
* [ ] **Issuer Registry** (IRS) and **CRL** published over HTTPS with valid signatures.

## RRC.8 QA Exit Criteria

* [ ] Zero **ERROR** diagnostics across CTM suite on reference validator.
* [ ] No open `TEMPORARY_LIMITATION` deviations in RPP for Level C.
* [ ] All WARNs are triaged; any remaining are documented in VCT and SI notes.

## RRC.9 Post-Release Monitoring

* [ ] Establish **badge verification** telemetry thresholds (optional, privacy-preserving).
* [ ] Define **revocation SLA** checks using SAS/CRL freshness windows.
